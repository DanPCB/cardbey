# Hero & Avatar Image – Where They Are Rendered (Code Reference)

Reference for the three UIs you asked about.

---

## 1. Draft Review  
**URL:** `http://localhost:5174/app/store/temp/review?mode=draft&jobId=...&generationRunId=...`

**Route:** `StoreReviewPage` → `StoreDraftReview` → `StoreReviewHero`

### Where hero/avatar URLs come from
- **StoreReviewPage** does not pass `initialHeroUrl` / `initialAvatarUrl` on draft (only on publish-review).
- **StoreDraftReview** computes hero and avatar inside the render that returns `<StoreReviewHero>`:
  - **Hero:** `getEffectiveHero()` → priority: `heroOverride` → `optimisticHero` → draft (via `resolveBrandImages`). Result is `heroImageUrl`.
  - **Avatar:** `resolveBrandImages(...).avatarUrl`, then (for publish-review only) `initialAvatarUrl` override. Result is `resolvedAvatar`.

**File:** `apps/dashboard/cardbey-marketing-dashboard/src/features/storeDraft/StoreDraftReview.tsx`  
- Hero resolution (getEffectiveHero): **~3547–3614**  
- Avatar resolution (resolvedAvatar): **~3925–3942**  
- Props passed to StoreReviewHero: **~3963–3971**  
  - `logoUrl={resolvedAvatar}`  
  - `heroImageUrl={heroImageUrl}`  

### Where the DOM is rendered
**File:** `apps/dashboard/cardbey-marketing-dashboard/src/features/storeDraft/review/StoreReviewHero.tsx`

- **Hero image (banner):**
```tsx
// Lines 96–112
<div className="relative w-full aspect-[21/9] max-h-48 rounded-t-xl overflow-hidden ...">
  {heroVideoUrl ? (
    <video src={heroVideoUrl} ... />
  ) : heroImageUrl ? (
    <img
      src={heroImageUrl}
      alt=""
      className="absolute inset-0 w-full h-full object-cover"
    />
  ) : null}
  ...
</div>
```

- **Avatar (logo):**
```tsx
// Lines 127–134
{logoUrl ? (
  <img
    src={logoUrl}
    alt={displayName}
    className="w-full h-full object-cover"
  />
) : (
  ...
)}
```

So for **draft review**, the image you see is generated by the `<img src={heroImageUrl} />` and `<img src={logoUrl} />` in **StoreReviewHero.tsx** (lines 107–111 and 130–134), with `heroImageUrl` and `logoUrl` coming from **StoreDraftReview.tsx** as above.

## 2. Publish-Review  
**URL:** `http://localhost:5174/app/store/temp/publish-review?jobId=...&heroUrl=...&avatarUrl=...`

**Route:** Same: `StoreReviewPage` → `StoreDraftReview` → `StoreReviewHero`

### Where hero/avatar URLs come from
- **StoreReviewPage** reads query and passes:
  - `initialHeroUrl = searchParams.get('heroUrl') || searchParams.get('visualUrl')`
  - `initialAvatarUrl = searchParams.get('avatarUrl')`

  **File:** `apps/dashboard/cardbey-marketing-dashboard/src/pages/store/StoreReviewPage.tsx`  
  **Lines:** 74–75

- **StoreDraftReview** uses these on publish-review (readonly):
  - **Hero:** In `getEffectiveHero()`, **first** priority when `readonly && initialHeroUrl`: use `initialHeroUrl` (from URL).  
    **File:** `StoreDraftReview.tsx` **~3549–3552**
  - **Avatar:** `resolvedAvatar = (readonly && initialAvatarUrl?.trim()) ? initialAvatarUrl.trim() : brandResolvedAvatar.avatarUrl`.  
    **File:** `StoreDraftReview.tsx` **~3941–3942**

So for **publish-review**, the same **StoreReviewHero** component is used; the only difference is that `heroImageUrl` and `logoUrl` are driven by URL params (`heroUrl`, `avatarUrl`) via `initialHeroUrl` / `initialAvatarUrl`.

### Where the DOM is rendered
Same as draft review: **StoreReviewHero.tsx**  
- Hero: **107–111** (`<img src={heroImageUrl} />`)  
- Avatar: **130–134** (`<img src={logoUrl} />`)

## 3. Public Storefront  
**URL:** `http://localhost:5174/preview/store/<storeId>?view=public`

**Route:** `StorePreviewPage` (public/minimal view)

### Where hero/avatar URLs come from
**File:** `apps/dashboard/cardbey-marketing-dashboard/src/pages/public/StorePreviewPage.tsx`

- **Resolvers (from store/preview data):**
  - `resolvedPublicHeroUrl = resolveStoreHeroImage(finalPreviewData)` — **~689–692**
  - `resolvedPublicAvatarUrl = resolveStoreAvatarImage(finalPreviewData)` — **~693–696**
- **Optional single override:** `visualUrlParam = searchParams.get('visualUrl')`
- **Final URLs:**
  - `publicHeroUrl = visualUrlParam || resolvedPublicHeroUrl` — **~699**
  - `publicAvatarUrl = visualUrlParam || resolvedPublicAvatarUrl` — **~700**

Resolver implementations: **`src/lib/storeMedia.ts`**  
- `resolveStoreHeroImage`: **~29–37**  
- `resolveStoreAvatarImage`: **~45–54**

### Where the DOM is rendered
**File:** `StorePreviewPage.tsx`

**Mobile layout:**
- **Hero (background):** **~1274–1283**
```tsx
<div
  className="w-full aspect-[21/9] max-h-40 md:max-h-48 ..."
  style={
    publicHeroUrl
      ? { backgroundImage: `url(${publicHeroUrl})` }
      : { backgroundColor: '#e5e7eb' }
  }
/>
```
- **Avatar (logo):** **~1289–1304**
```tsx
<img
  src={(isMinimalPublicView ? publicAvatarUrl : finalPreviewData.images?.[0])!}
  alt={finalPreviewData.storeName}
  className="w-full h-full object-cover rounded-2xl"
/>
```

**Desktop layout:**
- **Hero (background):** **~1630–1639** — same pattern, `style={{ backgroundImage: \`url(${publicHeroUrl})\` }}`
- **Avatar (logo):** **~1644–1659** — `<img src={... publicAvatarUrl ...} />`

So for **public storefront**, the hero image is generated by the `backgroundImage: url(publicHeroUrl)` divs and the avatar by the `<img src={publicAvatarUrl}>` in **StorePreviewPage.tsx** at the line ranges above.
