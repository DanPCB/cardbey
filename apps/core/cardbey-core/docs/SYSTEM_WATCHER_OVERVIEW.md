# System Watcher Overview

## Introduction

The System Watcher is an AI-powered monitoring and insight system that collects events from devices, orchestrator, and dashboard, analyzes them, and provides actionable insights and suggestions.

## Architecture

### Database Models

#### SystemEvent
Stores raw system events from various sources:
- `source`: "device" | "orchestrator" | "dashboard"
- `type`: Event type (e.g., "heartbeat", "playback_start", "error")
- `deviceId`: Optional device ID
- `tenantId`: Optional tenant ID
- `severity`: "info" | "low" | "medium" | "high" | "critical"
- `payload`: JSON data
- `createdAt`: Timestamp

#### SystemInsight
Stores analyzed insights generated by the watcher:
- `title`: Insight title
- `severity`: Severity level
- `category`: "devices" | "ai_pipelines" | "campaigns" | "performance" | "security" | "other"
- `summary`: Brief description
- `payload`: Full insight object as JSON
- `createdAt`: Timestamp

### Services

#### systemEventsService.ts
- `recordSystemEvent()`: Record a new system event
- `getRecentEvents()`: Query events with filters
- `computeAggregates()`: Compute statistics from events

#### systemInsightsService.ts
- `createSystemInsight()`: Save an insight
- `getRecentInsights()`: Query recent insights
- `mapInsightToWatcherFormat()`: Convert DB model to API format

### Orchestrator Entry

The `system_watcher` orchestrator entry:
- Location: `src/orchestrator/systemWatcher.ts`
- Function: `runSystemWatcher()`
- Uses TextEngine to analyze events and generate insights/suggestions
- Returns: `SystemWatcherResult` with summary, insights, and suggestions

### API Routes

#### POST /api/watcher/event
Record a system event.

**Request:**
```json
{
  "source": "device" | "orchestrator" | "dashboard",
  "type": "string",
  "severity": "info" | "low" | "medium" | "high" | "critical",
  "deviceId": "optional",
  "tenantId": "optional",
  "payload": {}
}
```

**Response:**
```json
{
  "ok": true
}
```

#### GET /api/watcher/insights
Get recent insights.

**Query params:**
- `limit`: Number of insights (default: 10)

**Response:**
```json
{
  "ok": true,
  "insights": [SystemWatcherInsight[]]
}
```

#### POST /api/watcher/chat
Ask the watcher a question.

**Request:**
```json
{
  "question": "string",
  "from": "ISO date string (optional)",
  "to": "ISO date string (optional)",
  "filters": {
    "source": "device" | "orchestrator" | "dashboard"
  }
}
```

**Response:**
```json
{
  "ok": true,
  "result": {
    "summary": "string",
    "insights": [SystemWatcherInsight[]],
    "suggestions": [SystemWatcherSuggestion[]]
  }
}
```

### Scheduled Jobs

The insight generation job runs every 10 minutes:
- Gets events from the last 10 minutes
- Computes aggregates
- Calls the orchestrator to analyze
- Saves insights to the database

Location: `src/scheduler/systemWatcherJob.ts`

## Dashboard Components

### Components

1. **InsightSeverityBadge**: Colored badge showing severity
2. **WatcherInsightCard**: Card displaying a single insight
3. **WatcherInsightFeed**: Feed of insights with auto-refresh
4. **WatcherChatPanel**: Chat interface for asking questions
5. **WatcherChatButton**: Floating button to open chat
6. **SystemWatcherPanel**: Panel wrapper for the insight feed

### Integration

- **WatcherChatButton**: Available globally in App.jsx (except login/frontscreen pages)
- **SystemWatcherPanel**: Can replace or supplement existing insights cards
- **WatcherInsightFeed**: Can be added to dashboard pages

### Usage

```tsx
import { SystemWatcherPanel, WatcherChatButton } from './components/watcher';

// In dashboard
<SystemWatcherPanel theme="dark" />

// Global chat button (already integrated)
<WatcherChatButton />
```

## Device Agent Integration

The DeviceAgent should POST events to `/api/watcher/event` for:
- Heartbeats: `{ source: "device", type: "heartbeat" }`
- Playback events: `{ source: "device", type: "playback_start" | "playback_end" }`
- Self-healing: `{ source: "device", type: "self_heal_action" }`
- Errors: `{ source: "device", type: "error", severity: "high" }`

## Types

All types are defined in `packages/ai-types/src/systemWatcher.ts`:
- `SystemWatcherInsightSeverity`
- `SystemWatcherInsight`
- `SystemWatcherSuggestion`
- `SystemWatcherResult`

## Migration

To enable the System Watcher:

1. Run migration:
```bash
cd apps/core/cardbey-core
npx prisma migrate dev --name add_system_watcher
```

2. The watcher will automatically:
   - Start the insight generation job on server startup
   - Register routes in Express
   - Be available via API endpoints

3. In the dashboard:
   - WatcherChatButton is already integrated globally
   - Add SystemWatcherPanel to desired pages

## Notes

- Events are limited to 500 per query to control costs
- Insights are generated every 10 minutes by default
- The chat interface queries events from the last 24 hours by default
- If no insights are available, the UI gracefully shows an empty state
