# JWT 401 Debug Fix Summary

## Problem
- GET `/api/auth/me` returns 401 even though Bearer token is sent
- GET `/api/store/:id/context` returns 401
- Backend logs show `hasAuthHeader: true` and `authHeaderPrefix: 'Bearer eyJhbGciOiJIU...'`
- Cookies are not used (`hasCookie: false`)

## Root Cause Analysis
The JWT verification is failing silently. Common causes:
1. **Secret mismatch**: Token signed with one secret, verified with another
2. **Token expired**: Token has expired but error not clearly reported
3. **Invalid signature**: Token signature doesn't match secret
4. **Missing JWT_SECRET**: Using default secret that doesn't match token signer

## Solution: Enhanced Debug Logging + Better Error Messages

### Changes Made

#### 1. JWT_SECRET Validation on Startup
**File:** `apps/core/cardbey-core/src/middleware/auth.js`

- Added startup validation that warns if `JWT_SECRET` is not set
- Logs secret source (env vs default) and length
- Helps identify secret mismatch issues early

#### 2. Detailed Token Inspection Before Verification
**File:** `apps/core/cardbey-core/src/middleware/auth.js`

- Decodes token (without verification) to inspect:
  - Header: `alg`, `kid`, `typ`
  - Payload: `userId`, `sub`, `iss`, `aud`, `exp`, `iat`
  - Expiration status
- Logs token info in debug mode (gated by `DEBUG=true` or dev mode)

#### 3. Detailed JWT Verification Error Handling
**File:** `apps/core/cardbey-core/src/middleware/auth.js`

- Wraps `jwt.verify()` in try-catch with detailed error logging
- Maps JWT errors to specific reason codes:
  - `invalid_signature` - Secret doesn't match
  - `expired` - Token has expired
  - `malformed_token` - Token format is invalid
  - `missing_signature` - Token missing signature
  - `not_active` - Token not yet valid (nbf)
- Returns structured error response with reason code

#### 4. Improved Error Responses
**File:** `apps/core/cardbey-core/src/middleware/auth.js`

- All 401 errors now include:
  - `ok: false`
  - `error: 'UNAUTHORIZED'`
  - `reason: 'invalid_signature' | 'expired' | 'malformed_token' | ...`
  - `message: <human-readable message>`
  - `debug: <detailed info>` (only in dev mode)

#### 5. Enhanced optionalAuth Error Handling
**File:** `apps/core/cardbey-core/src/middleware/auth.js`

- Added error logging for optionalAuth (non-blocking)
- Handles dev-admin-token in optionalAuth too

## How to Debug

### 1. Enable Debug Logging
Set environment variable:
```bash
DEBUG=true
```

Or ensure `NODE_ENV !== 'production'` (automatic in dev mode)

### 2. Check Startup Logs
Look for:
- `[Auth] ✅ JWT_SECRET is configured` - Good
- `[Auth] ⚠️  JWT_SECRET not set` - Problem: using default secret

### 3. Check Request Logs
When 401 occurs, look for:
- `[Auth] JWT verification failed` - Shows exact error
- Check `reason` field in error response

### 4. Common Issues and Fixes

#### Issue: `invalid_signature`
**Cause:** Token signed with different secret than verification
**Fix:**
- Ensure `JWT_SECRET` in `.env` matches the secret used to sign tokens
- Check if frontend is using a different secret
- Verify token wasn't generated by another service

#### Issue: `expired`
**Cause:** Token has expired
**Fix:**
- Check `exp` field in token payload
- Increase `JWT_EXPIRES_IN` if needed
- Re-login to get new token

#### Issue: `malformed_token`
**Cause:** Token format is invalid
**Fix:**
- Check if token is being truncated or modified
- Verify token is complete (not cut off)
- Check if token encoding is correct
