\# Cardbey MI-First Development Rule (NON-NEGOTIABLE)



\## Identity

Cardbey is a \*\*Business Agent with a Merged-Intelligence (MI) brain\*\*.

It is NOT a feature platform with AI utilities.



UI is a console.

MI is the operator.



---



\## Absolute Rules



\### 1. MI Is the Operator

All business-meaningful actions MUST be initiated, decided, or validated by MI.



Forbidden:

\- UI-only workflows for core business tasks

\- Manual-first flows for scalable operations

\- Feature logic that bypasses MI



Allowed:

\- Manual UI actions ONLY as override or correction paths



---



\### 2. No Feature Without an MI EntryPoint

Every feature MUST map to a registered MI EntryPoint.



No EntryPoint = Feature does not exist.



---



\### 3. MI Is the Single Source of Business Truth

Readiness, gaps, fixes, progress, and outcomes come from MI — never UI heuristics.



---



\### 4. Scale-First Rule

If a task does not scale to 300+ items, humans cannot be the primary executor.



Manual work = MI capability gap.



---



\### 5. UI Never Mutates Business State Directly

UI may:

\- Request actions

\- Display MI state

\- Apply overrides explicitly flagged as manual



UI may NOT:

\- Perform bulk fixes

\- Decide readiness

\- Replace MI judgment



---



\### 6. Drift Is Failure

If MI and UI diverge, UI must be rolled back immediately.



No patches.

No parallel flows.

No exceptions.



---



\## Internal Mantra

> We don’t build features.  

> We train a business mind.
