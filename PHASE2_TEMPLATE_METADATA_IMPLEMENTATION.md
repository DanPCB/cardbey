# Phase 2 - Template Metadata Implementation Summary

## Overview
Added template metadata fields to support business type and use case filtering for Phase 2 starter kits.

## Changes Made

### 1. Prisma Schema Updates
**File:** `apps/core/cardbey-core/prisma/schema.prisma`

Added three new optional JSON fields to the `CreativeTemplate` model:
- `businessCategories Json?` - Array of business types (e.g., `["cafe", "restaurant", "bakery"]`)
- `useCases Json?` - Array of use cases (e.g., `["hero", "menu", "promo", "social"]`)
- `styleTags Json?` - Array of style characteristics (e.g., `["modern", "playful", "minimal"]`)

Also updated the `orientation` field comment to include `"portrait" | "landscape"` in addition to existing values.

**Migration Required:**
```bash
npx prisma migrate dev --name add_template_metadata
```

### 3. API Route Updates
**File:** `apps/core/cardbey-core/src/routes/creativeTemplates.js`

#### POST `/api/creative-templates` (Create)
- Accepts optional `businessCategories`, `useCases`, and `styleTags` in request body
- Validates arrays and stores as JSON strings
- Sets to `null` if not provided (backward compatible)

#### PUT `/api/creative-templates/:id` (Update)
- Accepts optional `businessCategories`, `useCases`, and `styleTags` in request body
- Updates only if provided (partial update support)
- Sets to `null` if explicitly set to empty array

#### GET `/api/creative-templates` (List)
- Parses JSON metadata fields and returns as arrays
- Includes `businessCategories`, `useCases`, and `styleTags` in response
- Handles both string (JSON) and array formats for backward compatibility

## Usage in Phase 2

The metadata fields will be used to:
1. **Filter templates by business type**: Match `businessCategories` with `BusinessProfile.type`
2. **Filter by use case**: Select templates with specific `useCases` (e.g., "hero", "menu")
3. **Filter by style**: Match `styleTags` with `BusinessProfile.stylePreferences`
4. **Filter by orientation**: Match `orientation` with desired output format

Example query for Phase 2:
```typescript
const templates = await prisma.creativeTemplate.findMany({
  where: {
    isActive: true,
    isSystem: true,
    // Filter by business category
    businessCategories: {
      array_contains: businessProfile.type, // Pseudo-code - actual syntax depends on DB
    },
    // Filter by use case
    useCases: {
      array_contains: 'hero',
    },
  },
});
```

## Notes

- All metadata fields are optional to maintain backward compatibility
- JSON fields are used instead of native arrays for SQLite compatibility
- Templates without metadata will still work (they just won't be filtered in Phase 2)
- The seed script can be run multiple times (idempotent) to update existing templates
